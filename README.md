## Hey, I'm Barry

Software engineer in Quincy, MA. I write 6502 assembly for fun, Terraform for profit, and MCP servers because I can't stop connecting things to AI.

---

### What I'm Into

**Retro Computing** -- I build things for the Commodore 64. In assembly. By choice. A chess engine, a BASIC extension framework, a reusable assembly library, and hardware mods for the C128. I also built [a 6502 emulator with SID sound](https://github.com/barryw/NovaVM) and [a unit test CLI](https://github.com/barryw/sim6502) so I can write tests for 8-bit code like it's 2026.

**MCP Servers** -- I build bridges between AI agents and self-hosted services. So far: [Paperless-ngx](https://github.com/barryw/PaperlessMCP), [Immich](https://github.com/barryw/ImmichMCP), and [VICE](https://github.com/barryw/ViceMCP) (yes, I gave Claude access to a Commodore 64 emulator).

**Infrastructure** -- Terraform modules, Kubernetes, AWS. I've shipped Terraform providers for [PiHole](https://github.com/barryw/terraform-provider-pihole) and [Vaultwarden](https://github.com/barryw/terraform-provider-vaultwarden), plus modules for [Zookeeper on AWS](https://github.com/barryw/terraform-aws-zookeeper) and [RDS cost management](https://github.com/barryw/terraform-aws-rds-scheduler).

**Self-Hosted Everything** -- If there's a service I use daily, I've probably written tooling for it. Paperless-ngx, Immich, PiHole, Vaultwarden, Vikunja -- I run my own stack and build the glue to make it work the way I want.
